Overview

DreamSDK Shell is a component built on top on MSYS.

MSYS is a collection of GNU utilities such as bash, make, gawk and grep to allow building of applications and programs which depend on traditionally UNIX® tools to be present. It is intended to supplement MinGW and the deficiencies of the cmd shell.

The Shell component basically replaces the msys.bat file, provided in MinGW/MSYS standard installation, even if it is still there. This was made for two reasons:

  • You have the possibility to choose the Shell interface of your choice really easilly;
  • And you have the possibility to open the Shell in a specific directory.

To be clear, the DreamSDK Shell is only a launcher which starts the real MSYS Shell.

Installation details

Everything custom items related to the DreamSDK environment package are installed in the /opt directory. You can access to this directory by opening the Shell and entering the following:

cd /opt

In summary, every toolchains and KallistiOS components are in /opt/toolchains/dc/ directory when all DreamSDK binaries are in /opt/dreamsdk/ directory.

More precisely:

  • The SuperH toolchain is installed in /opt/toolchains/dc/sh-elf/.
  • The Yamaha AICA toolchain is installed in /opt/toolchains/dc/arm-eabi/.
  • The KallistiOS main library is installed in /opt/toolchains/dc/kos/.
  • The KallistiOS Ports collection is in /opt/toolchains/dc/kos-ports/.
  • The Dreamcast Tool sources (both for Serial and Internet Protocol) are in /opt/toolchains/dc/dcload/.
  • The Dreamcast Tool binaries are in /opt/toolchains/dc/bin/.
  • The DreamSDK helpers applications are in /opt/dreamsdk/.

Please note that the /opt/toolchains/dc/ directory is the standard location specified in the KallistiOS documentation. This directory location was used for the DreamSDK environment package to respect the standard documentation.

Configuration details

The DreamSDK configuration is located in /etc/dreamsdk/ directory, where the MSYS configuration is located in /etc/ directory, mainly in the /etc/fstab file which indicates all mount points used by your DreamSDK environment (i.e. the MSYS environment).

Usage details

So now that everything is setup what are the main steps to develop for the Sega Dreamcast? Basically the steps are:

  • You can starts from a KallistiOS example, which are stored in /opt/toolchains/dc/kos/examples/. This is a good idea as the Makefile of almost all KallistiOS samples are nice, clear and ready to be used.
  • Duplicate an example project from KallistiOS and copy it on your /home directory (or somewhere else).
  • Use Notepad++ or something else and develop your program.
  • Build it by entering the make command in the project's directory.
  • Execute it (if available) by entering the make run command.
  • When everything is done you can clean all files by entering make clean.

When your program is ready, you should be able to distribute it.

Open the Shell directly in a specific directory

If you pass a directory to the dreamsdk-shell.exe binary, the DreamSDK Shell will be opened directly in that directory.

Example: dreamsdk-shell C:\Temp will open the Shell in the /C/Temp directory.
