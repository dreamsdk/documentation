Components

The Components tab is useful for several things:

  • It displays the home base (root) directory. You have the possibility to open it in the Windows® Explorer using the Open button.
  • You can change the whole Sega Dreamcast toolchain used by DreamSDK, or just the GNU Debugger (GDB) package.
  • You can check the version of the main libraries (i.e., KallistiOS, KallistiOS Ports and Dreamcast Tool) used by DreamSDK.

About changing the Components configuration

When installing DreamSDK, you have the choice to install the Stable toolchain, based on GCC 13 (not compatible with Windows® XP), the Old Stable toolchain, based on GCC 9, or the Legacy toolchain, based on GCC 4.

Later on, after installation, you have the possibility to change the toolchain components using DreamSDK Manager through this tab. When changing these, the whole toolchain will be rebuilt (excluding the KallistiOS Ports). Please note, you can only have one version of the toolchain installed at once, so you will have to choose between GCC 4, 9 or 13. Usually, GCC 13 is better as it's more modern (of course).

You also have the option to change the GNU Debugger (GDB) package version only, basically if you want to change the Python revision support. Indeed, all GNU Debugger (GDB) packages provided with DreamSDK are the same but they have been built using different Python versions. You have of course the choice to disable completely Python extensions by selecting the value Disable Python for GDB. Please note, for enabling Python extensions, Python 32-bit build needs to be installed.

To apply a new toolchain and/or another GNU Debugger (GDB) package, you just have to:

  1. Click on Change Toolchain or Change GNU Debugger (GDB) radio buttons;
  2. Click on Toolchain and/or GNU Debugger (GDB) drop-down lists;
  3. Click on the Apply button.

Note: if you don't click on the Apply button after doing the changes, the fields will be reset to their initial values.
