Frequently Asked Questions (FAQ)

Q: What is DreamSDK?

A: It's a package which contains everything you need to develop for the Sega Dreamcast on Microsoft Windows. This package is called the DreamSDK environment.

Q: Why another package? Again?

A: Because this one is of course better! More seriously, this package is different from the others since it includes only absolutely required binaries plus the two required precompiled toolchains (SH-4 and ARM). It doesn't contains KallistiOS itself, which is handled by DreamSDK Manager. Doing so will allow the DreamSDK package and your local installation staying always up-to-date.

Q: What is KallistiOS?

A: It's the software library which handles the Sega Dreamcast hardware. It's a great way to develop freely for your Sega Dreamcast. This library is not included in the DreamSDK package, instead it's handled by the DreamSDK Manager tool, part of the DreamSDK environment.

Q: What is KallistiOS Ports?

A: It's a port collection made by the KallistiOS team. Ports are additional libraries to extend KallistiOS, e.g. adding PNG, JPEG or SDL support to your program.

Q: And... what is Dreamcast Tool?

A: It's the program responsible of sending your program to the Dreamcast through the serial (Coders Cable) cable or via your Broadband/LAN Adapter (if you have the luck to have one), in order to execute/debug them. Dreamcast Tool, also called dc-tool, is the server part. The Sega Dreamcast part is called dcload, it exists in two version, serial and ip.

Q: I found a bug or a missing software. How can I contact you?

A: Please visit the DreamSDK Website and use the Contact form.

Q: It's really mandatory to use the DreamSDK Shell to build Dreamcast programs?

A: It's better, but not mandatory. Check the DreamSDK Shell documentation to learn the possibilities.

Q: Can I use the package with an IDE (Integrated development environment)?

A: Since all Sega Dreamcast development tools (e.g. toolchains binaries) have been statically linked, you can use them outside the DreamSDK environment. This hasn't been tested however. By the way this should be a side project upon the DreamSDK package!

Q: Can I contribute to the DreamSDK project? The first thing I want to submit is a fix for that horrible Frenglish!

A: Of course! The project is hosted on GitHub. Feel free to send a Pull Request!
