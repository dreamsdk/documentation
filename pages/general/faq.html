Frequently Asked Questions (FAQ)

Q: What is DreamSDK?

A: It's a package which contains everything you need to develop for the Sega Dreamcast on Microsoft Windows. This package is called the DreamSDK environment.

Q: What is NOT DreamSDK?

A: It's not a easy-to-use framework like Unity. You will need to learn Sega Dreamcast programming and especially KallistiOS libraries capabilities. In clear, DreamSDK is just a ready-to-use environment that allow you to compile/build Sega Dreamcast programs, but won't help you to learn and/or facilitate Sega Dreamcast programming.

Q: Why another package? Again?

A: Because this one is of course better! More seriously, this package is different from the others since it includes only absolutely required binaries plus the two required precompiled toolchains (SH-4 and ARM) in order to build Sega Dreamcast programs. It doesn't contains KallistiOS itself, which is handled by DreamSDK Manager. Doing so will allow the DreamSDK package and your local installation staying always up-to-date, without the need of downloading an updated DreamSDK package.

Q: What is a homebrew program?

A: It's a program for your Sega Dreamcast which is made by you. Read Wikipedia for more information.

Q: What is an .elf file?

A: It's the binary format used for programs for the Sega Dreamcast (in our case). It's the exact equivalent to .exe programs on Windows. Executable Linkable Format (.elf) programs made with the DreamSDK environment are suitable for the Sega Dreamcast.

Q: What is a selfboot (or autoboot/bootable) disc?

A: It's a special disc image format which allow the Sega Dreamcast to run the software on it, like a real game disc. There is a big vulnerability in the BIOS code of the Sega Dreamcast, but to summarize, this vulnerability allows the boot of burned CD-R like genuine discs if these discs are burned in a special way. This kind of discs are usually called selfboot discs. Click here to learn more.

Q: What is an IP.BIN file?

A: IP stands for Initial Program. It's the bootstrap used on CD-R images. This little program tell the Sega Dreamcast the name of the boot program (usually 1ST_READ.BIN, see below) plus other informations. This file is not in the file system but it's inserted in the first CD-R sectors. If you select the IPCreate program when installing DreamSDK, you'll be able to generate these kind of files.

Q: What is a scrambled program?

A: Scrambled programs are .elf programs which are converted into 1ST_READ.BIN files. This program binary format is a special format allowing the boot of your program from a CD-R. In clear, the CD-R making process can be summarized as:

Basically, from your source directory, you build the main binary program by entering the make command. Then, you can enter the make dist command to generate the raw binary. Then finalize the process by scrambling the program by just typing scramble yourprog.bin 1ST_READ.BIN. Now, you just have to generate an IP.BIN file and you are good to go: put everything on a disc image. The whole process is described here.

Q: What is KallistiOS?

A: KallistiOS (or KOS) is the software library which allows access to the Sega Dreamcast hardware. It's a great way to develop freely for your Sega Dreamcast. This library is not included in the DreamSDK package, instead it's handled by the DreamSDK Manager tool, part of the DreamSDK environment. Basically, KallistiOS will be downloaded by the DreamSDK Manager tool but it isn't embedded in the DreamSDK Setup program.

Q: What is KallistiOS Ports?

A: It's a port collection made by the KallistiOS team. Ports are additional libraries to extend KallistiOS, e.g. adding PNG, JPEG or SDL support to your program.

Q: And... what is Dreamcast Tool?

A: It's the program responsible of sending your program to the Dreamcast through the serial (Coders Cable) cable or via your Broadband/LAN Adapter (if you have the luck to have one), in order to execute/debug them. Dreamcast Tool, also called dc-tool, is the server part. The Sega Dreamcast part is called dcload and exists in two version, serial and ip. Click here to learn everything on it.

Q: It's really mandatory to use the DreamSDK Shell to build Dreamcast programs?

A: It's better, but not mandatory. Check the DreamSDK Runner documentation or even better, use the Code::Blocks IDE.

Q: Can I use the package with an IDE (Integrated development environment)?

A: Since DreamSDK R2, the environment is fully compatible with Code::Blocks 17.12 stable release. Just install the IDE and select the appropriate option when installing DreamSDK. Code::Blocks was chosen because it's free, extensible, open source and nice to use, and even better, it's still compatible with the old Windows XP. There is no plan to support other IDE yet.

Q: From the DreamSDK Shell, how can I access to another directory which is NOT in the DreamSDK tree? Is it mandatory to create my Sega Dreamcast homebrew projects in the DreamSDK environment, i.e. in my /home directory from the DreamSDK environment?

A: No. You can access to all Windows drives by entering something like cd /<drive_letter>/<my_folder1>/<my_folder2>/, e.g. cd /C/Dreamcast/My/Project/, or you can just pass the directory to open to the dreamsdk.exe binary.

Q: How can I test all the environment? Is there any way to test if everything is working correctly in my environment?

A: Yes. The simplest way to check that everything is working is compiling all KallistiOS samples. But before that, you have to install all KallistiOS Ports. After that, just type cd /opt/toolchains/dc/kos/examples/ then enter make. All examples should compile fine, and all .elf programs will be generated. But really, it should be not necessary to do that, plus, the generation time is really long.

Q: I found a bug or a missing software. How can I contact you?

A: Please visit the DreamSDK Website or even better, you can post your issue on the official KallistiOS forum.
