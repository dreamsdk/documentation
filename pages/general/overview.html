DreamSDK Overview

DreamSDK is a modern, ready-to-use environment for Sega Dreamcast development, designed for the Microsoft Windows platform.

It's a big, self-contained package embedding all needed software to freely develop for the Sega Dreamcast system on Windows. This package is built on open-source software only thus it doesn't contains any copyrighted software. See the licenses section for more information.

In clear, DreamSDK provides you:

  • An UNIX® like environment;
  • Three prebuilt toolchains;
  • Some side utilities in order to manage the whole environment.

Architecture summary

As described in the introduction, DreamSDK is a package which is built upon a lot of components.

The schema below describes the architecture of the whole environment:

MinGW/MSYS

The UNIX® like environment is provided by MinGW/MSYS. Basically this is the environment base, which provides UNIX® tools on Windows.

Everything is built upon this middleware.

DreamSDK Shell

DreamSDK Shell is a component built on the top of MSYS. Basically, this is the main program of this environment.

Toolchains

The toolchains are basically the set of the tools needed to produces your programs, i.e. compiling programs and debugging them. In our case, these toolchains are used to compile and build binary code targeting the Sega Dreamcast.

Three toolchains are provided by the DreamSDK environment:

  • For the SuperH CPU processor (sh-elf), the main processor of the Sega Dreamcast;
  • For the Yamaha AICA processor (arm-eabi), the sound processor. You don't have to use it directly, as this is needed only to compile the sound driver included in KallistiOS (see below).
  • The last one is the original Win32 toolchain (mingw32) provided by MinGW/MSYS, this allow you to produces Windows binaries, e.g. compiling KallistiOS utilities like vqenc, scramble, dc-tool... But, you have the possibility to develop pure Windows binaries as well for other purposes if you need to.

All the toolchains are assembled from several components:

  • Binutils
  • GNU Compiler Collection (GCC)
  • Newlib
  • GNU Debugger (GDB)

These components are already prebuilt and installed with the DreamSDK environment.

KallistiOS

KallistiOS provides you a ready-to-use software library to interact with the Sega Dreamcast hardware. This facilitates a lot the development, instead of writing low-level code to access the Sega Dreamcast hardware: everything is already done by this library.

That's why DreamSDK uses KallistiOS (KOS), which is the best option today for Sega Dreamcast development. By the way, it has been used to produce multiple commercial quality games for sale, like Feet of Fury, Maqiupai, Inhabitants, Cool Herders, Dynamite Dreams and many more. DreamSDK respects all the standard KOS prerequisites and requirements, that means this environment is 100% compliant with KallistiOS.

KallistiOS Ports

KallistiOS Ports are ports collection made by the KallistiOS team. Ports are additional libraries used to extend the features of KallistiOS, like e.g. adding support for PNG, JPEG or SDL to your program.

Dreamcast Tool

Dreamcast Tool, or dc-tool, is the tool used to send your programs to the Sega Dreamcast in order to execute and debug them.

This program is split in two parts, dcload for the server part (executed on the Sega Dreamcast) and dc-tool for the client part (executed from your computer).

Depending of your connection kind, you have to use the correct program version:

  • If you own the Serial cable (aka the Coders Cable), the version to use is dcload-serial;
  • If you have the luck owning an expensive Broadband Adapter or even the LAN Adapter, which are Ethernet cards for the Sega Dreamcast, you will have to use the dcload-ip version.

This component is disabled by default, as you have the possibility to execute your programs directly through a Dreamcast emulator, i.e. using nullDC, Demul, Makaron, etc.

Managing KallistiOS with DreamSDK environment

Instead of the majority of previous released packages of this kind (e.g. dcfreedev — with that really out-dated GCC 3.4.6 —, DC Dev Environment (DDE), Code::Blocks DC Development...), DreamSDK only includes the prebuilt toolchains but not the KallistiOS library itself: KallistiOS is handled by the DreamSDK Manager application.

This is the reason why DreamSDK ask you to be connected to Internet when installing it: KOS is installed automatically at the end of the installation process. If you have cancelled the process or the process has failed, just fire-up DreamSDK Manager again and that application will (re)install KOS if needed.

Alternatives to KallistiOS

As you saw, DreamSDK uses KallistiOS, as this software library is the most advanced library for Sega Dreamcast development.

The other alternative to KOS is Libronin. This software library was used for making DreamSNES. It should work under DreamSDK, even if this scenario was not tested.

You are free to use other libraries if you want, but KOS is a great choice if you are new to Dreamcast development.

DreamSDK Manager

DreamSDK Manager is the application which handle everything of the DreamSDK environment. Basically this application was made to update KallistiOS but it can also handle KallistiOS Ports even Dreamcast Tool configuration.
