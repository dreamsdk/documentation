DreamSDK Overview

DreamSDK is a modern, ready-to-use environment for Sega Dreamcast development, designed for the Microsoft Windows platform.

It's a big, self-contained package embedding all needed software to freely develop for the Sega Dreamcast system on Windows. This package is built on open-source software only thus it doesn't contains any copyrighted software. See the licenses section for more information.

In clear, DreamSDK provides you:

  • An UNIX® like environment;
  • Two pre-built toolchains for the Dreamcast, each targetting differents processors:
    • The SuperH CPU (the main processor);
    • The Yamaha AICA (the sound processor).
  • Some side utilities in order to manage the whole environment.

All the sub-sections below describes in details the environment composition.

In more details, the DreamSDK environment package is composed by several components:

  • The UNIX® like environment is provided by MinGW/MSYS;
  • All the toolchains are assembled by Binutils, GNU Compiler Collection (GCC), Newlib and GNU Debugger (GDB);
  • Plus, some additional components made specially for the DreamSDK environment:
    • DreamSDK Manager, which handles the KallistiOS library;
    • DreamSDK Shell, a component which is built on the top of MSYS.

Keep reading the document to learn what is KallistiOS and how it's managed in DreamSDK.

KallistiOS

DreamSDK uses KallistiOS (KOS), which is the best option for Dreamcast development. KOS is a pro-grade development system for the Dreamcast. It has been used to produce multiple commercial quality games for sale, like Feet of Fury, Maqiupai, Inhabitants, Cool Herders, Dynamite Dreams...

In clear, KallistiOS provides you a ready-to-use software library to interact with the Dreamcast hardware.

DreamSDK respects all the standard KOS prerequisites and requirements.

Managing KallistiOS

Instead of the majority of previous released packages of this kind (e.g. dcfreedev — with that out-dated GCC 3.4.6 —, DC Dev Environment (DDE), Code::Blocks DC Development...), DreamSDK only includes the pre-built toolchains (both for the SuperH CPU and Yamaha AICA) but not the KallistiOS library itself.

Instead, KallistiOS is handled by the DreamSDK Manager application.

This is the reason why DreamSDK asks the user to be connected to Internet; KOS is installed automatically at the end of the installation. If you have cancelled the process or the process has failed, just fire-up DreamSDK Manager and the application will (re)install KOS if needed.

Alternatives to KallistiOS

DreamSDK uses by default KallistiOS, as this software library is the most advanced library for Sega Dreamcast development.

The other alternative is Libronin. This software library was used for making DreamSNES. It should work under DreamSDK, even if this scenario was not tested.
