Windows Integration

DreamSDK Shell is an interesting component as it allows usage of the DreamSDK environment outside it!

With this component, you have the possibility to execute commands inside the MSYS Shell without actually opening it.

Supposing your homebrew program directory is located in D:\Dreamcast\MyProgram\ and you are using a standard KallistiOS Makefile structure (i.e. look to the /opt/toolchains/dc/kos/examples/dreamcast/ directory for templates).

Create a new batch file in that D:\Dreamcast\MyProgram\ directory and call it build.cmd. Copy this content on it:

@dreamsdk "make"

Now, double-click to run it: You will have your program built, exactly if you had manually entered that make command in the MSYS Shell!

Of course, this will work only if you have added DreamSDK in your PATH variable at the installation! If you have not updated the PATH variable, you will need to put the full path to the dreamsdk binary, e.g. ${DREAMSDK_HOME}\msys\1.0\opt\dreamsdk\dreamsdk.exe.

Now, you can even do the same for the clean target of your Makefile, by creating a clean.cmd file and putting this content on it:

@dreamsdk "make clean"

So the result will be:

Double-click on that clean.cmd batch file in your D:\Dreamcast\MyProgram\ directory to execute the make clean command directly on the current directory, where that script is located.

Of course you can pass any command to the DreamSDK Shell, but usually, it's a great feature used only for make commands.
