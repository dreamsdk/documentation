Code::Blocks 17.12 Integration

Starting from DreamSDK R2, you have the possibility to use the Code::Blocks IDE in the top of the DreamSDK package!

This will help you to use all the possibilities of the Sega Dreamcast development on Windows, in a really user-friendly way.

Please select a chapter below:

  • Prerequisites
  • Installation
  • Overview
  • Creating a Sega Dreamcast Project

Prerequisites

If you want to use the Code::Blocks IDE integration, you need to:

  • Install the Code::Blocks IDE Ver. 17.12 release package. You have the choice to use the package form you prefer (e.g. you may install the embedded MinGW version or the standalone version only). The only critical prerequisite is the version revision: it should be the 17.12 stable release, i.e. the latest official build as today (February 2019). Code::Blocks needs to be installed before starting the DreamSDK installation.
  • After installing the Code::Blocks IDE, run it at least once. This will generate the default.conf file, which will be patched by the DreamSDK installation program, as some parameters need to be written in it.
  • Optional: If you want to use the Code::Blocks debug capabilities, you will need to have either a Coders Cable, a LAN Adapter or a Broadband Adapter (best choice).

If you have respected the prerequisites, you may now startup the DreamSDK installation.

Installation

When running the DreamSDK installation, at the Select Components step, you should select the Code::Blocks 17.12 IDE integration option.

This will add a new step of the DreamSDK installation which will allow you to select:

  • The Code::Blocks installation directory (usually %ProgramFiles%\CodeBlocks\) ;
  • The Code::Blocks profile file (usually %AppData%\CodeBlocks\default.conf).

Of course this will work only if you have respected the prerequisites!

Overview

After installing the Code::Blocks Integration, your Code::Blocks installation will have several new features, made for the DreamSDK package. This includes, but not limited to:

  • The Debugger plugin will run a loader before trying to connect to the target (i.e. Dreamcast Tool will be run before GDB) ;
  • The Compiler plugin will run a loader instead of trying to run the compiled program (again, Dreamcast Tool will run the compiled program on the target).
  • The GNU GCC Compiler for Sega Dreamcast profile will be installed.
  • The Sega Dreamcast Project wizard will be installed.

All these features are available only if you are installing the 17.12 release. It won't work with any other release!

Creating a Sega Dreamcast Project

Before using the Sega Dreamcast Project wizard, it would be a great idea to execute the DreamSDK Manager utility at least once, in order to install additional libraries. These additional libraries are managed throught the KallistiOS Ports tab.

If you don't install any KallistiOS Ports libraries, you will have at least 4 libraries installed anyway:

  • libkosext2fs
  • libkosutils
  • math
  • libppp

libkosext2fs, libkosutils and libppp are installed along KallistiOS, when math (aka libm) is installed with Newlib. That's why all these libraries are always installed, even if you don't select any additional KallistiOS Ports.
